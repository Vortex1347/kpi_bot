# PROJECT

## Цель

Разработать внутреннего Telegram-бота для KPI-оценки сотрудников (5–15 участников) с автоматической генерацией Excel-отчетов для руководителя.

## Функциональные решения (утверждено)

- Stack: `NestJS + TypeScript + Telegram Bot API + Vite React (results dashboard)`.
- Frontend: только CRM-приложение (`/login`, `/results`, `/crm`), отдельный лендинг не используется.
- Хранилище: `PostgreSQL + Prisma`.
- Отчет: `Excel (.xlsx)`.
- Анонимность: сотрудники не видят чужие ответы; руководитель видит все.
- Регистрация сотрудников: в боте через `/start` (пошагово: ФИО, затем отдел).
- Telegram-меню сотрудника:
  - для незарегистрированного пользователя по `telegramId` показывается кнопка `Регистрация`,
  - после регистрации доступны кнопки `Заполнить KPI` и `Редактировать профиль`.
  - после completion анкеты доступна inline-кнопка `Перезаполнить анкету` на 10 минут.
  - callback-кнопки анкеты защищены от повторного клика (кнопка скрывается сразу после нажатия).
- Напоминания: отключены.
- Бонус: не рассчитывается (поле фиксировано как `0`).
- Закрытие кампании: вручную (через CRM-кнопку `Закрыть KPI`, команда `/close_kpi` остается fallback).
- Руководитель: один (`LEAD_TELEGRAM_ID` в `.env`).
- Локальный мониторинг результатов: web страница `/results` (таблица + действия руководителя).
- Авторизация CRM: вход обязателен перед доступом к `/results`.
- Роль `SUPERVISOR`: обязательна для кнопок старта/закрытия/экспорта KPI в CRM.
- Помесячная аналитика KPI: `GET /results/statistics` + таблица трендов в вебке.
- Web action-кнопки в CRM:
  - Старт KPI-кампании,
  - Закрытие KPI-кампании,
  - Экспорт Excel.
- При старте KPI в CRM выбирается месяц периода (`YYYY-MM`), который сохраняется в кампании как `assessmentMonth`.

## KPI-модель

- 5 вопросов с оценками: `100 / 90 / 80 / 70 / 60 or less` (внутреннее числовое значение для `60 or less` — `0`).
- Раздел 1 = 70% (внутри 70/30).
- Раздел 2 = 30% (внутри 60/20/20).
- Эффективные веса вопросов: `49, 21, 18, 6, 6`.

## Команды

Сотрудник:

- `/start`
- кнопка `Заполнить KPI` (в Telegram-меню)
- кнопка `Перезаполнить анкету` (в течение 10 минут после completion)
- `/fill_kpi` (резервно)

Руководитель:

- `/start_kpi`
- `/start_kpi YYYY-MM` (optional month fallback)
- `/close_kpi`
- `/generate_reports`

## Текущий статус

### Выполнено

- Backend-каркас NestJS в `backend/`.
- Модули backend: `auth`, `users`, `bot`, `employee`, `survey`, `evaluation`, `kpi`, `report`, `results`, `prisma`.
- Frontend feature: `modules/results` для таблицы KPI-ответов.
- Помесячная статистика и тренд сотрудников за период `1..36` месяцев.
- UI-кнопки действий:
  - "Старт KPI",
  - "Закрыть KPI",
  - "Экспорт в Excel".
- Новая Prisma-схема + SQL-миграция.
- Seed KPI-вопросов.
- Генерация `summary` и персональных Excel-отчетов.
- Актуальные unit-тесты для `health`, `kpi`, `results`.

### В бэклоге

- Команды редактирования ответов руководителем внутри бота (если понадобится).
- PDF-экспорт (опционально).
- Тонкая настройка текстов и UX в Telegram.

## Критерий готовности текущего этапа

- Сотрудник может зарегистрироваться и пройти опрос через кнопки.
- Сотрудник может обновить ФИО/отдел через кнопку `Редактировать профиль`.
- Сотрудник может перезаполнить анкету в течение 10 минут после completion.
- Доступ к `/results` идет через CRM-login, а action-кнопки доступны только SUPERVISOR.
- При запуске KPI фиксируется выбранный период месяца, а статистика строится по нему.
- KPI можно собирать каждый месяц и видеть динамику по месяцам/сотрудникам.
- Руководитель может открыть/закрыть кампанию и получить отчеты.
- Проект собирается и тесты проходят.
