# TESTING

## Backend test stack

- Unit tests: `vitest`
- Type check/build: `tsc`

## Команды

```bash
cd backend
npm run build
npm test
```

```bash
cd ../frontend
npm run build
npm test
npm run test:e2e -- --grep "crm smoke"
```

Примечание для e2e: использовать `localhost` (не `127.0.0.1`), чтобы refresh-cookie CRM auth корректно работал в браузерных сценариях.

## Текущий набор unit-тестов

- `health.service`:
  - liveness response
  - readiness success
  - readiness db-failure
- `kpi.service`:
  - расчет KPI по весам с учетом неотвеченных вопросов
- `evaluation.service`:
  - окно перезаполнения анкеты (10 минут после completion)
  - reset ответов при перезаполнении
  - блокировка прямого изменения ответов после completion без перезаполнения
- `results.service`:
  - пустой ответ при отсутствии кампаний
  - агрегация строк таблицы с KPI-расчетом
  - 404 при невалидном `campaignId`
  - monthly-агрегация по последним месяцам
  - тренд сотрудника между месяцами
  - валидация `months` (1..36)
  - web action start-campaign (success)
  - web action close-campaign (success)
  - web action export-excel (генерация summary + employee files)

## Manual smoke (Telegram)

1. Открыть CRM login (`/login`) и войти SUPERVISOR.
2. Сотрудник отправляет `/start`, видит кнопку `Регистрация`, проходит шаги (ФИО -> отдел), после чего видит кнопки `Заполнить KPI` и `Редактировать профиль`.
3. Проверить кнопку `Редактировать профиль`: обновить ФИО/отдел и убедиться, что профиль сохранился.
4. В CRM (`/results`) выбрать месяц KPI (`YYYY-MM`) и нажать `Старт KPI`.
5. Сотрудник в Telegram нажимает кнопку `Заполнить KPI` и отвечает на 5 вопросов.
6. Проверить, что низкая оценка отображается как `60 or less` (вместо `0`) в Telegram и в таблице CRM.
7. Проверить anti-double-click: после нажатия варианта ответа кнопки текущего вопроса скрываются и не принимают повторный клик.
8. После завершения проверить появление кнопки `Перезаполнить анкету`, нажать ее в течение 10 минут и пройти анкету заново.
9. Проверить, что после нажатия `Перезаполнить анкету` кнопка скрывается и не запускается повторно.
10. Проверить, что после истечения 10 минут перезаполнение недоступно.
11. В CRM проверить таблицу ответов и фильтр по месяцу.
12. Проверить блок "Статистика по месяцам" и "Динамика сотрудников".
13. В CRM нажать `Закрыть KPI`.
14. В CRM нажать `Экспорт в Excel` и проверить пути к файлам.
